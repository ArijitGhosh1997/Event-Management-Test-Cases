# FEATURE_006: Add Single Registered User Button in Event Registered Tab

## Module
Event Management → Submit Event → Registered Tab

---

## Summary
Introduce a "Single Registered User" button within the Registered Tab to allow quick access to individual registered user details.

---

## Description
A new button should be added in the Registered Tab under the Submit Event section. This button will allow admins to view or interact with individual registered user details efficiently.

---

## Key Features

### Button Placement
- Located under Registered Tab
- Positioned near existing actions (e.g., event-related options)
- Maintain proper spacing and alignment

---

### Button Design
- Follow existing UI design guidelines
- Clearly labeled (e.g., "View Registered User")
- Visually consistent with other buttons

---

### Functionality
- On click, display registered user details
- Allow interaction such as:
  - View details
  - Edit information (if applicable)
  - Trigger related actions

---

### Edge Case Handling
- If no users are registered:
  - Show appropriate message (e.g., "No users found")

---

### Responsive Behavior
- Button should work across:
  - Desktop
  - Tablet
  - Mobile devices

---

## Expected Behavior
- Button is visible and properly aligned  
- Clicking button displays user details  
- System handles empty state gracefully  

---

## Acceptance Criteria
- Button is added in Registered Tab  
- UI matches existing design patterns  
- Functionality works correctly  
- Responsive across devices  

---

Feature
